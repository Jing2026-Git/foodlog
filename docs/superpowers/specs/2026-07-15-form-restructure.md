# 饭记 · 表单重构设计

> 日期：2026-07-15
> 状态：已确认

## 背景

重构餐记录表单，将字段按功能模块分组（基础信息/食物信息/营养估算/自定义标签），增加多选下拉列表（Chips 标签式），实现 AI 智能识别和自动填充，优化用户体验。

## 信息架构

### 表单四模块结构

```
┌─────────────────────────────────────────────┐
│  📅 Part1: 基础信息（必填）                 │
│  ├─ 日期* │ 时间* │ 餐次* │ 来源*         │
│  │         时间自动匹配餐次                  │
├─────────────────────────────────────────────┤
│  🍽️ Part2: 食物信息                        │
│  ├─ 食物名称* │ 商家名 │ 价格              │
│  │         全部 AI 识别提取                 │
├─────────────────────────────────────────────┤
│  🔥 Part3: 营养估算                        │
│  ├─ 热量 │ 蛋白质 │ 碳水 │ 脂肪 │ 分量     │
│  │         AI 根据图片内容估算              │
│  │         分量=减半时自动减半数值          │
├─────────────────────────────────────────────┤
│  🏷️ Part4: 自定义标签（可折叠）             │
│  ├─ 口味 │ 主食 │ 蛋白 │ 蔬菜 │ 饮品 │ 评价 │
│  │         Chips 多选，AI 识别自动填充       │
│  │         选饮品时显示"饮品配方"输入框      │
└─────────────────────────────────────────────┘
```

## Part1: 基础信息

### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 日期 | 日期选择器 | 是 | 默认今天 |
| 时间 | 时间输入 | 是 | 默认当前时间，自动匹配餐次 |
| 餐次 | Chips 下拉 | 是 | 根据时间自动推荐，可手动调整 |
| 来源 | Chips 下拉 | 是 | 外卖/剩饭/自炊/大餐 |

### 时间自动匹配餐次规则

```
05:00 - 11:00    → 早餐
11:00 - 14:00    → 午餐
14:00 - 17:00    → 下午加餐
17:00 - 20:00    → 晚餐
20:00 - 23:00    → 晚上加餐
```

**加餐灵活性**：用户可在任何时间段手动选择"加餐"，系统不会自动覆盖。时间匹配仅作为默认推荐。

### 来源选项

| 值 | 显示 | 说明 |
|----|------|------|
| takeout | 外卖 | 外卖平台点餐 |
| leftover | 剩饭 | 剩菜，分量自动设为减半 |
| home | 自炊 | 自己做饭 |
| feast | 大餐 | 下馆子/聚会 |

## Part2: 食物信息

### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 食物名称 | 文本输入 | 是 | AI 识别截图内容 |
| 商家名 | 文本输入（条件显示） | 否 | AI 识别商家名，仅来源=外卖时显示 |
| 价格 | 数字输入 | 否 | AI 识别截图价格 |

### AI 识别

截图上传时，AI 自动识别并填充：
- 食物名称（OCR + 图像识别）
- 商家名（OCR 识别商家 Logo/名称）
- 价格（OCR 识别金额）

## Part3: 营养估算

### 字段定义

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 热量 | 数字输入 | 否 | AI 根据图片内容估算 |
| 蛋白质 | 数字输入 | 否 | AI 根据图片内容估算 |
| 碳水 | 数字输入 | 否 | AI 根据图片内容估算 |
| 脂肪 | 数字输入 | 否 | AI 根据图片内容估算 |
| 分量 | 下拉选择 | 否 | 正常/减半 |

### 分量联动逻辑

```
来源 = "剩饭" → 分量自动设为"减半" → 营养数值自动减半
分量手动改为"减半" → 营养数值自动减半
分量改为"正常" → 营养数值恢复原值（如果之前是减半状态）
```

## Part4: 自定义标签

### 字段定义（Chips 多选）

| 字段 | 选项 | 说明 |
|------|------|------|
| 口味 | 清淡、正常、重口 | 描述食物口味 |
| 主食 | 低GI粗粮、精米精面 | 主食类型 |
| 蛋白 | 牛肉、羊肉、猪肉、禽肉、鱼虾、豆制品 | 蛋白质来源 |
| 蔬菜 | 深色叶子菜、菌菇、瓜果 | 蔬菜类型 |
| 饮品 | 奶茶、咖啡、果茶、纯茶 | 饮品类型 |
| 评价 | 健康餐、需改善、含糖、高油、高钠、高辣 | 健康评价标签 |

### 折叠交互

- **新增记录**：默认折叠，点击标题展开
- **编辑记录**：如果已有标签则自动展开

### 饮品配方（条件显示）

选择"饮品"标签后，显示"饮品配方"输入框：
```
饮品配方：珍珠奶茶去冰三分糖
```

### AI 自动填充

AI 识别时根据图片内容自动选择标签，用户可手动调整。

## 数据模型变更

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| merchant | string | 商家名 |
| drink_formula | string | 饮品配方 |
| flavor | string[] | 口味标签数组 |
| staple | string[] | 主食标签数组 |
| protein | string[] | 蛋白标签数组 |
| vegetable | string[] | 蔬菜标签数组 |
| drink | string[] | 饮品标签数组 |
| evaluation | string[] | 评价标签数组 |

### 字段映射（向后兼容）

旧字段 `tags` 不再使用，改为结构化标签字段。旧数据迁移：
- `taste` → `flavor`
- `staple_food` → `staple`
- `meat_type` → `protein`
- `vegetable_type` → `vegetable`
- `food_category` → 根据值映射到 `drink` 或其他

## AI Prompt 调整

增强识别指令，要求返回：
1. 食物名称、商家名、价格
2. 营养估算（根据图片内容）
3. 结构化标签（口味、主食、蛋白、蔬菜、饮品、评价）
